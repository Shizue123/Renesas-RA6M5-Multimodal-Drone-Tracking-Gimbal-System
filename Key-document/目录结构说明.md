## 1. `Drone_Tracking_RA6M5/` — RA6M5 主控固件

这是整个系统的**控制核心**，基于瑞萨 e² studio + FSP 框架开发。

```
Drone_Tracking_RA6M5/
├── configuration.xml          — FSP 图形化配置文件（UART、引脚、时钟）
├── Drone_Tracking_RA6M5 Debug_Flat.launch  — 调试启动配置
├── ra_cfg.txt                — RA 配置摘要
│
├── src/                       — ★ 应用层源码（开发者最常改的地方）
│   ├── hal_entry.c            — 主事件循环：UART 中断、LED 状态、遥测发送、舵机初始化
│   ├── hal_warmstart.c        — 热启动处理
│   ├── openmv_drv.c/h         — OpenMV 视觉帧接收驱动（8 字节协议解析，双缓冲）
│   ├── control/               — 控制算法
│   │   ├── pid_controller.c/h      — 增量式 PID（IIR 滤波、死区、积分分离）
│   │   └── tracking_controller.c/h — 四态追踪状态机、雷达 Kalman、多模融合、增益调度
│   └── drivers/               — 硬件驱动
│       ├── ld2450_driver.c/h       — LD2450 毫米波雷达帧解析（30 字节协议 + IIR 平滑）
│       └── st3215_driver.c/h       — ST-3215 舵机通信驱动（飞特协议、半双工、SYNC_WRITE）
│
├── ra_gen/                    — FSP 自动生成代码（Generate Project Content 产出）
│   ├── main.c                 — 入口：调用 hal_entry()
│   ├── hal_data.c/h           — UART 实例配置（g_uart2/3/5/9 的波特率、回调、引脚）
│   ├── common_data.c/h        — 通用外设初始化
│   ├── vector_data.c/h        — 中断向量表
│   ├── pin_data.c             — 引脚复用配置
│   └── bsp_clock_cfg.h        — 时钟配置
│
├── ra_cfg/                    — FSP 模块配置头文件
│   └── fsp_cfg/
│       ├── bsp/               — BSP 配置（芯片型号、引脚、OFS）
│       ├── r_ioport_cfg.h     — IO 端口配置
│       ├── r_iwdt_cfg.h       — 看门狗配置
│       └── r_sci_uart_cfg.h   — SCI UART 配置
│
├── ra/                        — FSP 框架源码（一般不需要修改）
│   ├── arm/CMSIS_6/           — ARM CMSIS 核心头文件
│   └── fsp/
│       ├── inc/               — FSP API 头文件（r_uart_api.h 等）
│       └── src/
│           ├── bsp/           — BSP 实现（启动、时钟、中断、IO）
│           ├── r_sci_uart/    — SCI UART 驱动实现
│           ├── r_ioport/      — IO 端口驱动
│           └── r_iwdt/        — 看门狗驱动
│
├── openmv_scripts/            — OpenMV 机载脚本（部署到 OpenMV 的正本）
│   └── main_tracking.py       — FOMO 目标检测 + MA + Kalman + 8 字节帧发送
│
├── script/
│   └── fsp.ld                 — FSP 链接脚本
│
├── Debug/                     — 编译输出（可再生，不要手动编辑）
│   ├── *.elf.in, *.map, *.srec, *.rpd  — 固件二进制
│   ├── compile_commands.json   — 编译数据库（clangd 代码补全用）
│   ├── fsp_gen.ld, memory_regions.ld  — 生成的链接脚本
│   ├── makefile               — 构建系统
│   └── ra_gen/, src/, ra/     — 各模块的 .o/.d 编译中间文件
│
└── .settings/                 — e² studio IDE 首选项（不需要手动编辑）
```

### 开发者重点关注

| 你想做什么 | 去哪里 |
|-----------|--------|
| 改追踪算法/状态机 | `src/control/tracking_controller.c` |
| 调 PID 参数 | `src/control/tracking_controller.c` 里的默认初始化，或 `pid_controller.c` |
| 改雷达解析逻辑 | `src/drivers/ld2450_driver.c` |
| 改舵机通信/限位 | `src/drivers/st3215_driver.c` |
| 改视觉帧协议 | `src/openmv_drv.c` + `openmv_scripts/main_tracking.py` 两端同步 |
| 改遥测 JSON 格式 | `src/hal_entry.c` 里的 sprintf |
| 改 UART 引脚/波特率 | `configuration.xml` → 重新 Generate → 查看 `ra_gen/hal_data.c` |

## 2. `Drone_Tracking_WebCloud/` — 云端监控看板

```
Drone_Tracking_WebCloud/
├── main.py              — FastAPI 后端（JWT认证、RBAC、WebSocket、REST API）
```
