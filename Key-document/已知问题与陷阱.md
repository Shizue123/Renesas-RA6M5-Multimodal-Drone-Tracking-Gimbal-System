> 以下内容基于 2026/3/30 对全部源码的逐文件审计得出。后续修复后请标注状态。

## 设计层面的不一致

### 1. OpenMV 与 RA6M5 帧中心假设不匹配

- **OpenMV 侧**（`main_tracking.py:19-20`）：实际分辨率 96×96，中心 = (48, 48)，发送的是相对于 48 的偏移量
- **RA6M5 侧**（`openmv_drv.h:30-31`）：`OPENMV_FRAME_CENTER_X/Y = 120`，遥测中 `target_x/y` 以 120 为中心

**为什么当前能工作**：帧传输的是偏移量（`cx - 48`），RA6M5 收到偏移后直接用于 PID，不依赖绝对坐标。`OPENMV_FRAME_CENTER_X/Y = 120` 仅在遥测 JSON 和角度估算中使用。

**何时会出问题**：
- 更换 OpenMV 分辨率（如改用 160×160）→ 必须同步改 `CENTER_X/Y` 和 `OPENMV_FOV_PX`
- 依赖遥测中 `target_x/y` 做二次分析时，绝对值不代表真实像素位置

### 2. VISION_COASTING 恢复路径依赖雷达

- **设计意图**：视觉短暂丢失时靠惯性滑行，恢复后回到精瞄
- **实际行为**（`tracking_controller.c`）：`VISION_COASTING` → `VISION_INNER_LOOP` 的恢复需要**视觉+雷达同时有效**且角度门限通过
- **后果**：如果雷达掉线或被遮挡，纯视觉恢复被阻断，系统会在 800ms 后回退到 `SEARCHING`，重新走雷达确认流程

### 3. 视觉锁定存在 3 帧延迟

- OpenMV 端 `LOCK_THRESHOLD_FRAMES = 3`（`main_tracking.py:23`），前 2 帧报 `STATUS_LOST`
- RA6M5 融合代码只在 `status == VISION_STATUS_LOCKED` 时采信视觉
- **实际影响**：视觉生效有 ~60ms 延迟（假设 ~30fps），快速穿越场景可能来不及锁定

### 4. OpenMV 候选目标回退策略

- 当连续性判定（`TRACKING_RADIUS = 24px`）失败时，OpenMV 会回退选择帧内**最高置信度**的检测结果
- **风险**：如果画面中存在多个相似物体，可能跳到错误目标

## 硬件与调试陷阱

### 7. Tilt 天顶极性翻转

- 当 Tilt 位置 < 1023（`TILT_ZENITH_POS`）时，摄像头视角翻转
- 代码中已处理：Pan 误差符号在天顶以下自动反转
- **但**：如果手动将 `TILT_PHYS_MIN_POS` 改到 1023 以下，必须确认极性翻转逻辑仍正确

### 8. P402 DIR 引脚冗余

- URT-1 扩展板自带硬件自动流向控制
- `hal_entry.c` 中仍手动翻转 P402（TX 前拉高，TX 完成后拉低）作为安全冗余
- **不要移除此逻辑**，在某些 URT-1 版本上硬件自动控制可能不可靠

### 9. 舵机初始化顺序不可跳步

正确顺序：
```
Ping → 关扭矩 → 归中(2047) → 开扭矩
```
- 如果跳过"关扭矩"直接归中，舵机可能在非中心位突然锁定，导致机械冲击
- 如果跳过 Ping，无法检测舵机是否在线，后续指令静默失败

### 10. 舵机 Slew Rate 限制

- 最大 ±23°/帧（约 22.7 servo steps @ 50Hz）
- 超过此限制会触发 clamp，导致追踪响应变慢但避免机械应力
- 如果目标高速移动追不上，**不要**简单增大 slew limit，应先检查 PID 增益和前馈参数

### 11. ESP32 WiFi 凭据硬编码

- `esp32_async_gateway.ino:8-9` 中 WiFi SSID/密码直接写在源码里
- 换场地时必须改源码重新编译上传
- 云端地址 `CLOUD_URL` 同理

### 12. WebCloud JWT 密钥默认随机

- `main.py:44` 中 `JWT_SECRET` 默认使用 `secrets.token_urlsafe(48)` 随机生成
- **每次重启服务**，所有已发出的 token 失效，用户需重新登录
- 生产环境应通过环境变量 `JWT_SECRET` 固定密钥（见 `.env.example`）

## 调参经验

### PID 调参注意事项

- **先调 Pan 再调 Tilt**：Tilt 物理范围小（1080~2457），更易过冲
- **死区 = 8px 是经验值**：太小会导致中心抖动，太大会导致精度下降
- **积分分离阈值**：误差很大时关闭积分，避免 windup；默认 Pan=60px, Tilt=50px
- **悬停模式**：误差 < 15px 时 Kp 降至 0.25 倍、Kd 归零，防止微幅振荡
- 详细调参方法论：→ [Key-document/PID调制.md](PID调制.md)

### 雷达前馈

- 角速度前馈 `ω = Vx / Y`（目标横向速度 / 纵深）
- 近距离时 Y 很小，ω 会爆炸 → 已用 `FUSION_DEPTH_MIN_MM = 100mm` 钳位
- 如果近距离追踪震荡，检查 `near_gain_scale`（默认 0.4）是否足够小
