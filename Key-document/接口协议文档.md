# 接口协议文档 — 四路 UART 帧格式与状态机

> 本文档基于当前代码审计生成，所有字段均标注源码出处。

## 1. OpenMV 视觉帧（RA6M5 ← OpenMV）

**物理层**：UART9 / SCI9，115200 8N1  
**源码**：`openmv_drv.c/h`，OpenMV 侧 `main_tracking.py`

### 帧格式（8 字节，大端序）

| 字节偏移 | 字段 | 值/范围 | 说明 |
|---------|------|--------|------|
| 0 | Header1 | `0x55` | 固定帧头 |
| 1 | Header2 | `0xAA` | 固定帧头 |
| 2 | Status | 0-3 | 见状态码表 |
| 3 | X_H | uint8 | 目标 X 偏移高字节 |
| 4 | X_L | uint8 | 目标 X 偏移低字节 |
| 5 | Y_H | uint8 | 目标 Y 偏移高字节 |
| 6 | Y_L | uint8 | 目标 Y 偏移低字节 |
| 7 | Checksum | uint8 | `(Byte0+Byte1+...+Byte6) & 0xFF` |

### 坐标定义

- OpenMV 脚本内部：96×96 像素，中心 (48, 48)，发送的是相对中心的偏移
- RA6M5 接收端：`openmv_drv.h` 定义 `OPENMV_FRAME_CENTER_X/Y = 120`，将收到的偏移量加上 120 后使用
- **注意**：两端中心假设不同（48 vs 120），当前靠偏移量传输规避了此差异；若需换分辨率必须两端同步修改

### Status 状态码

| 值 | 枚举名 | 含义 | 来源 |
|---|--------|------|------|
| 0 | `VISION_STATUS_LOST` | 目标丢失 | `openmv_drv.h:34` |
| 1 | `VISION_STATUS_LOCKED` | 目标锁定 | `openmv_drv.h:35` |
| 2 | `VISION_STATUS_PREDICTED` | 预测滑行中 | `openmv_drv.h:36` |
| 3 | `VISION_STATUS_HEARTBEAT` | 心跳（无目标但在线） | `openmv_drv.h:37` |

### 接收端解析

- **Lock-free 双缓冲**：ISR 写 `buffer[1-idx]`，主循环读 `buffer[idx]`，原子翻转
- **超时**：500ms 无新帧 → `COMM_STATUS_TIMEOUT`（`openmv_drv.h:32`）
- **帧错误**：50ms 字节级超时触发状态机复位

## 2. LD2450 雷达帧（RA6M5 ← LD2450）

**物理层**：UART2 / SCI2，256000 8N1  
**源码**：`ld2450_driver.c/h`

### 帧格式（30 字节）

| 字节偏移 | 字段 | 值 | 说明 |
|---------|------|-----|------|
| 0 | Header1 | `0xAA` | 固定帧头 |
| 1 | Header2 | `0xFF` | 固定帧头 |
| 2-3 | Type | uint16 LE | 帧类型标识 |
| 4-11 | Target1 | 8 字节 | 目标 1 数据 |
| 12-19 | Target2 | 8 字节 | 目标 2 数据 |
| 20-27 | Target3 | 8 字节 | 目标 3 数据 |
| 28 | Footer1 | `0x55` | 固定帧尾 |
| 29 | Footer2 | `0xCC` | 固定帧尾 |

### 单目标数据结构（8 字节，小端序）

| 偏移 | 字段 | 类型 | 单位 | 说明 |
|-----|------|------|------|------|
| 0-1 | X | int16 LE | mm | 水平位置（+ 右，− 左） |
| 2-3 | Y | uint16 − 32768 | mm | 纵深（0 ~ 6000mm） |
| 4-5 | Vx | int16 LE | cm/s | 速度（+ 远离，− 接近） |
| 6-7 | Reserved | — | — | 保留 |

### 接收端解析状态机

```
IDLE → 收到 0xAA → WAIT_HEADER2
WAIT_HEADER2 → 收到 0xFF → RECV_DATA (等待 26 字节)
RECV_DATA → 26 字节齐 → WAIT_FOOTER1
WAIT_FOOTER1 → 收到 0x55 → WAIT_FOOTER2
WAIT_FOOTER2 → 收到 0xCC → 帧有效 → IDLE
         任意步骤超时 50ms → 复位到 IDLE
```

来源：`ld2450_driver.c:20-26`（`LD2450_ParseState_e` 枚举）

### 后处理

- **IIR 平滑**：`α = 0.3`（`ld2450_driver.h:54`），`out = α * new + (1-α) * old`
- **Kalman 滤波**：恒速(CV)模型在 `tracking_controller.c` 中实现（`RadarCVKalman_t`），用于角速度前馈

## 3. ST-3215 舵机帧（RA6M5 ↔ ST3215）

**物理层**：UART5 / SCI5，1000000 8N1，半双工（URT-1 自动方向控制，P402 作冗余）  
**源码**：`st3215_driver.c/h`  
**协议**：飞特标准串行通信协议

### 通用帧格式

| 字节偏移 | 字段 | 值 | 说明 |
|---------|------|-----|------|
| 0 | Header1 | `0xFF` | 固定帧头 |
| 1 | Header2 | `0xFF` | 固定帧头 |
| 2 | ID | 1-253 | 舵机 ID（Pan=1, Tilt=2） |
| 3 | Length | uint8 | 后续字节数（含指令+参数+校验） |
| 4 | Instruction | uint8 | 指令码 |
| 5~N-1 | Parameters | — | 指令参数 |
| N | Checksum | uint8 | `~(ID + Length + Instruction + Params) & 0xFF` |

### 指令码

| 码 | 宏名 | 用途 | 来源 |
|---|------|------|------|
| `0x01` | `ST3215_CMD_PING` | 检测舵机在线 | `st3215_driver.h:66` |
| `0x02` | `ST3215_CMD_READ` | 读寄存器 | `st3215_driver.h:67` |
| `0x03` | `ST3215_CMD_WRITE` | 写寄存器 | `st3215_driver.h:68` |
| `0x04` | `ST3215_CMD_REG_WRITE` | 预写寄存器 | `st3215_driver.h:69` |
| `0x05` | `ST3215_CMD_ACTION` | 触发预写指令 | `st3215_driver.h:70` |
| `0x83` | `ST3215_CMD_SYNC_WRITE` | 同步写（广播） | `st3215_driver.h:71` |

### 关键寄存器

| 地址 | 宏名 | 长度 | 说明 | 来源 |
|-----|------|------|------|------|
| `0x28` | `ADDR_TORQUE_ENABLE` | 1B | 扭矩开关（0=关, 1=开） | `st3215_driver.h:76` |
| `0x2A` | `ADDR_TARGET_POSITION` | 2B LE | 目标位置 [0, 4095] | `st3215_driver.h:77` |
| `0x2C` | `ADDR_RUNNING_TIME` | 2B LE | 运动时间(ms) | `st3215_driver.h:78` |
| `0x38` | `ADDR_PRESENT_POSITION` | 2B LE | 当前位置 | `st3215_driver.h:79` |
| `0x3A` | `ADDR_PRESENT_SPEED` | 2B LE | 当前速度 | `st3215_driver.h:80` |
| `0x3C` | `ADDR_PRESENT_LOAD` | 2B LE | 当前负载 | `st3215_driver.h:81` |
| `0x3E` | `ADDR_PRESENT_VOLTAGE` | 1B | 当前电压 | `st3215_driver.h:82` |
| `0x3F` | `ADDR_PRESENT_TEMPERATURE` | 1B | 当前温度 | `st3215_driver.h:83` |

### 舵机物理限位

| 参数 | 值 | 来源 |
|------|-----|------|
| Pan 范围 | 0 ~ 4095 | `st3215_driver.h:57,60` |
| Tilt 安全范围 | 1080 ~ 2457 | `st3215_driver.h:53-54` |
| 天顶位 | 1023 | `tracking_controller.h:74` |
| 中心位 | 2047 | `st3215_driver.h:50` |
| 命令频率 | 50Hz（20ms 周期） | `tracking_controller.c:11` |

### 初始化顺序

```
1. Ping(ID=1) + Ping(ID=2) → 确认在线
2. Write(0x28, 0) → 关闭扭矩
3. SyncWrite(0x2A, 2047, 2047) → 归中
4. Write(0x28, 1) → 开启扭矩
```

不可跳步，否则舵机可能在非中心位突然锁定。

## 4. ESP32 遥测帧（RA6M5 → ESP32）

**物理层**：UART3 / SCI3，115200 8N1（交叉接线：RA6M5 TX → ESP32 RX）  
**源码**：`hal_entry.c:470-480`（发送端），`esp32_async_gateway.ino`（接收端）

### JSON 帧格式

```json
{"state":"VISION_LOCK","target_angle":2047,"vision_active":1,"tracking_mode":"VISION_INNER_LOOP","hover_active":0,"radar_dist":1500,"target_x":120,"target_y":115,"gimbal_tilt":2047}
```

每行一个 JSON 对象，以 `\n` 结尾。

### 字段定义

| 字段 | 类型 | 范围 | 说明 |
|------|------|------|------|
| `state` | string | `"SEARCH"` / `"RADAR_SLEW"` / `"VISION_LOCK"` / `"COASTING"` | 追踪状态简称 |
| `target_angle` | uint | 0 ~ 4095 | Pan 舵机当前位置 |
| `vision_active` | uint | 0 / 1 | 视觉通信是否正常 |
| `tracking_mode` | string | 状态机枚举名 | 详细追踪模式名 |
| `hover_active` | uint | 0 / 1 | 悬停缓追模式是否激活 |
| `radar_dist` | int | 0 ~ 6000 | 雷达测距(mm)，clamp 后 |
| `target_x` | int | 0 ~ 239 | 视觉目标 X 坐标，clamp 后 |
| `target_y` | int | 0 ~ 239 | 视觉目标 Y 坐标，clamp 后 |
| `gimbal_tilt` | uint | 1080 ~ 2457 | Tilt 舵机当前位置 |

### ESP32 接收与上报

- 接收：Serial2（GPIO16 RX, GPIO17 TX），逐行读取 JSON
- 上报：HTTP POST 到 `CLOUD_URL`（默认 `http://10.220.189.61:8000/api/device/upload`）
- 上报间隔：≥80ms，HTTP 超时 800ms
- 错误退避：连续 20 次失败后暂停 5s
- WiFi 断线重连：每 5s 尝试一次

## 5. 追踪状态机

### 四态定义

| 枚举值 | 名称 | 含义 | 来源 |
|-------|------|------|------|
| 0 | `STATE_SEARCHING` | 阶梯扫描搜索 | `tracking_controller.h:87` |
| 1 | `STATE_RADAR_OUTER_LOOP` | 雷达粗瞄，转向目标方位 | `tracking_controller.h:88` |
| 2 | `STATE_VISION_INNER_LOOP` | 视觉精瞄，PID 闭环跟踪 | `tracking_controller.h:89` |
| 3 | `STATE_VISION_COASTING` | 视觉丢失，惯性滑行 | `tracking_controller.h:90` |

### 转移条件

```
SEARCHING ──────────────────────────────────────────────────┐
  │                                                         │
  ├─ 雷达检测到目标 ──→ RADAR_OUTER_LOOP                     │
  │                                                         │
  └─ 视觉直接锁定 ───→ VISION_INNER_LOOP                     │
                                                             │
RADAR_OUTER_LOOP                                             │
  │                                                         │
  ├─ 视觉确认 (连续 3 帧 + 角度差 < 15°) ─→ VISION_INNER_LOOP│
  │                                                         │
  └─ 雷达丢失 ────────────────────────────→ SEARCHING ←──────┘
                                              ↑
VISION_INNER_LOOP                             │
  │                                           │
  └─ 视觉丢失 > 1500ms ──→ VISION_COASTING    │
                              │                │
                              ├─ 视觉恢复 ───→ VISION_INNER_LOOP
                              │
                              ├─ 雷达恢复 (滑行 < 800ms) ──→ RADAR_OUTER_LOOP
                              │
                              └─ 双丢失 > 800ms ──→ SEARCHING
```
