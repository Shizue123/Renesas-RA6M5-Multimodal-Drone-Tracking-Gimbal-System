# RA6M5 云台追踪系统 — 上板串口联调测试清单

> **版本**: v3.0 — 与 4 路 UART + LD2450 雷达 + ESP32 网关架构对齐  
> **目标硬件**: 瑞萨 RA6M5 + OpenMV H7 Plus + 飞特 ST-3215 ×2 + LD2450 毫米波雷达 + ESP32 网关  
> **视觉协议**: 8 字节大端 `[0x55][0xAA][Status][X_H][X_L][Y_H][Y_L][Checksum]`

---

## 一、当前 FSP 配置状态

`configuration.xml` 中已配置 **4 路 UART**:

| FSP 实例  | 用途                        | 波特率     | C 代码回调名          | SCI 通道 | 引脚                  |
|----------|----------------------------|-----------|----------------------|---------|----------------------|
| g_uart9  | OpenMV 视觉                | 115200    | `uart9_callback`     | SCI9    | P601=RXD9, P602=TXD9 |
| g_uart5  | ST-3215 舵机 (via URT-1)   | 1000000   | `uart5_callback`     | SCI5    | P501=TXD5, P502=RXD5 |
| g_uart2  | LD2450 毫米波雷达           | 256000    | `uart2_callback`     | SCI2    | P301=RXD2, P302=TXD2 |
| g_uart3  | ESP32 网关 / 调参终端       | 115200    | `uart3_callback` (运行时注入) | SCI3 | P706=RX3, P707=TX3 |

> `vector_data.c` 应包含 SCI9 / SCI5 / SCI2 / SCI3 各 4 个中断向量 (RXI/TXI/TEI/ERI)，共 **16 个中断向量**。

## 三、串口助手 Hex 测试帧 (模拟 OpenMV 下行数据)

将 USB-TTL 的 TX 连接到 RA6M5 的 **P601 (RXD9)**，GND 接 GND。  
串口参数: **115200, 8N1, 无流控**。

---

### 测试帧 1: 目标在画面正中心 — 唤醒追踪 (LOCKED, error=0,0)

```
55 AA 01 00 00 00 00 00
```

| 字节 | 值   | 含义                                |
|------|------|-------------------------------------|
| 0    | 0x55 | 帧头 1                              |
| 1    | 0xAA | 帧头 2                              |
| 2    | 0x01 | Status = LOCKED (1)                 |
| 3    | 0x00 | X_H = 0                             |
| 4    | 0x00 | X_L = 0 → error_x = 0              |
| 5    | 0x00 | Y_H = 0                             |
| 6    | 0x00 | Y_L = 0 → error_y = 0              |
| 7    | 0x00 | Checksum = (0x55+0xAA+0x01) & 0xFF  |

**预期物理反馈**: 两轴舵机不转动，保持当前位置。因误差=0 落入死区 (PID deadzone=8px)，PID 输出为零。手触舵机应**感到保持扭力（抵抗旋转）**但无主动运动。追踪进入 `STATE_VISION_INNER_LOOP`（视觉精瞄）。

---

### 测试帧 2: 目标在右下方 — 测试双轴联动 (LOCKED, error=+50,+50)

```
55 AA 01 00 32 00 32 64
```

| 字节 | 值   | 含义                                              |
|------|------|--------------------------------------------------|
| 2    | 0x01 | Status = LOCKED                                  |
| 3-4  | 00 32| error_x = +50 (目标偏右 50 像素)                  |
| 5-6  | 00 32| error_y = +50 (目标偏下 50 像素)                  |
| 7    | 0x64 | Checksum = (0x55+0xAA+0x01+0x00+0x32+0x00+0x32)  |

**预期物理反馈**: 云台 **Pan 轴 (水平) 右转** + **Tilt 轴 (俯仰) 下转**，SYNC_WRITE 广播帧驱动两轴同步运动。
- 每次收到帧，PID 计算增量，舵机位置 += delta
- 快速连续发送此帧，舵机会逐步偏离中心位 (2047)
- 手施力阻挡舵机旋转时，应感到**明显的抵抗扭力 (12V 供电)**
- 追踪状态为 `STATE_VISION_INNER_LOOP`

---

### 测试帧 3: 目标丢失 — 测试盲滑与归中 (LOST, error=0,0)

```
55 AA 00 00 00 00 00 FF
```

| 字节 | 值   | 含义                              |
|------|------|----------------------------------|
| 2    | 0x00 | Status = LOST (0)                |
| 3-6  | 全 0  | error = 0 (丢失时无意义)          |
| 7    | 0xFF | Checksum = (0x55+0xAA+0x00) & 0xFF |

**预期物理反馈**:
1. 若前一状态为 `STATE_VISION_INNER_LOOP`: 切换到 `STATE_VISION_COASTING`（盲滑），基于最后已知速度惯性滑动
2. 持续 **800ms** 盲滑超时后: 切换到 `STATE_SEARCHING`（搜索），两轴舵机 **归中到 position=2047** 并开始扫描
3. 归中后手触舵机仍感到保持扭力 (扭矩未释放, torque=ON)

## 四、补充测试帧

### 4.1 左上方负偏差 (LOCKED, error=-30,-20)

```
55 AA 01 FF E2 FF EC CC
```

X=-30 → 0xFFE2 (补码), Y=-20 → 0xFFEC  
校验和: `(0x55+0xAA+0x01+0xFF+0xE2+0xFF+0xEC) & 0xFF = 0xCC` ✓

### 4.2 预测帧 (PREDICTED, error=+10,-5)

```
55 AA 02 00 0A FF FB 05
```

X=+10 → 0x000A, Y=-5 → 0xFFFB  
校验和: `(0x55+0xAA+0x02+0x00+0x0A+0xFF+0xFB) & 0xFF = 0x05` ✓

## 五、联调推荐步骤

| 步骤 | 操作                                 | 预期物理反馈                              |
|------|--------------------------------------|------------------------------------------|
| 1    | 上电 (12V 供电 + USB 调试)            | 两轴舵机同时归中 (position=2047), 约 1 秒  |
| 2    | 等待归中完成, 手触舵机                  | 应感到保持扭力 (扭矩已使能)                |
| 3    | 串口发送 **帧 1** (00 00)             | 云台不动 (误差=0 → 死区 → 精瞄锁定)       |
| 4    | 连续快速发送 **帧 2** (+50 +50)       | 云台向右下转动                             |
| 5    | 停止发送 (等待 800ms 超时)            | 状态: 精瞄 → 盲滑 → 搜索(归中+扫描)       |
| 6    | 发送 **帧 3** (LOST)                 | 确认触发盲滑流程                           |
| 7    | 快速交替发送帧 1 和帧 2               | 云台在中心和偏移位之间来回追踪              |

## 六、追踪状态机 (当前架构)

```
STATE_SEARCHING  ─── 雷达检测到目标 ───→  STATE_RADAR_OUTER_LOOP
       ↑                                         │
       │                                  视觉锁定目标
       │                                         ↓
  超时/丢失 ←── STATE_VISION_COASTING ←──  STATE_VISION_INNER_LOOP
                   (盲滑 800ms)                   ↑
                                                  │
                                           视觉重新锁定
```

| 状态                      | 含义         | 触发条件                   |
|--------------------------|--------------|---------------------------|
| `STATE_SEARCHING`         | 全局搜索/复位 | 上电 / 盲滑超时 / 丢失超时  |
| `STATE_RADAR_OUTER_LOOP`  | 雷达粗瞄     | 雷达检测到有效目标          |
| `STATE_VISION_INNER_LOOP` | 视觉精瞄     | 视觉锁定 (status=LOCKED)   |
| `STATE_VISION_COASTING`   | 视觉盲滑     | 精瞄中视觉丢失 → 惯性滑动   |

## 七、故障排查速查表

| 现象                        | 可能原因                                                  |
|----------------------------|----------------------------------------------------------|
| OpenMV 数据收不到            | TX/RX 接反; 波特率不匹配; g_uart9 回调=NULL 未生效          |
| uart9_callback 不被调用      | FSP 中 g_uart9 的 Callback 字段未填写; channel 不是 9      |
| uart5_callback 不被调用      | FSP 中 g_uart5 的 Callback 字段未填写; SCI5 Pin 未映射     |
| uart2_callback 不被调用      | FSP 中 g_uart2 的 Callback 字段未填写; SCI2 Pin 未映射     |
| 舵机不动                     | 12V 未接; DIP 开关 ID≠1/2; 1Mbps 波特率不匹配             |
| 只有一个舵机动                | SYNC_WRITE 帧错误; 确认两个舵机 ID (1 和 2) 正确           |
| 舵机抖动/震荡                 | PID kp 过大; 舵机物理堵转; deadzone 太小 (当前=8px)        |
| 校验和报错 (checksum_errors) | 帧格式不对; 电平转换器信号质量差; 波特率误差超 2%           |
| 雷达数据收不到               | LD2450 接线反; 256000 波特率不匹配; SCI2 Pin 未映射        |
| ESP32 调参命令无响应          | g_uart3 运行时回调未注入; SCI3 Pin 未映射; 115200 不匹配   |
